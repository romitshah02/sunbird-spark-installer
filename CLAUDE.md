## What This Repo Does

`sunbird-spark-installer` automates deployment of the full Sunbird-ED learning platform on Kubernetes (Azure AKS or GCP GKE). It combines OpenTofu (Terraform-compatible) infrastructure provisioning with Helm chart deployments across 7 modular "building blocks".

## Key Commands

All deployments run from `opentofu/<provider>/<env-name>/` after copying the template:

```bash
cp -r opentofu/azure/template opentofu/azure/<env-name>
cd opentofu/azure/<env-name>

./install.sh                              # Full install (all phases)
./install.sh create_tf_backend            # Init OpenTofu state backend
./install.sh create_tf_resources          # Provision cloud infra (AKS/GKE, storage, etc.)
./install.sh install_helm_components      # Deploy all 7 Helm bundles
./install.sh install_component <name>     # Deploy a single bundle (e.g., edbb, learnbb)
./install.sh install_service <bundle> <chart>  # Deploy/upgrade single subchart (e.g., edbb player)
./install.sh certificate_config           # Inject RSA cert keys into registry service
./install.sh dns_mapping                  # Wait for DNS propagation (20-min timeout)
./install.sh generate_postman_env         # Build env.json for Postman tests
./install.sh run_post_install             # Run Postman collection for API validation
./install.sh destroy_tf_resources         # Tear down all cloud infra
```

For addons:
```bash
cd addons/<addon-name>/script
./addon.sh install [azure|gcp]
./addon.sh uninstall [azure|gcp]
```

## Architecture

### Building Blocks (Helm Umbrella Charts)

Installation order matters â€” each bundle builds on the previous:

| Order | Bundle | Key Contents |
|---|---|---|
| 1 | `monitoring` | Prometheus, Grafana, Loki, Grafana Alloy |
| 2 | `edbb` | Kong API Gateway, NGINX ingress, Kafka, Player portal, knowledge-mw |
| 3 | `learnbb` | Lern service, Keycloak, YugabyteDB, Elasticsearch, Redis, Flink jobs, cert services |
| 4 | `knowledgebb` | Knowlg service, Search, JanusGraph (on YugabyteDB), Flink jobs |
| 5 | `obsrvbb` | Telemetry service, Superset dashboards |
| 6 | `inquirybb` | Inquiry service |
| 7 | `additional` | cert-ng, NL web, volume autoscaler |

Each bundle is an umbrella Helm chart in `helmcharts/<bundle>/` with subcharts in `helmcharts/<bundle>/charts/`.

### Helm Value Layering

Every `helm upgrade --install` merges these files in order (later overrides earlier):
1. `helmcharts/<bundle>/values.yaml` â€” defaults
2. `helmcharts/<bundle>/ed-values.yaml` â€” education-specific overrides (if present)
3. `helmcharts/images.yaml` â€” all Docker image refs
4. `helmcharts/global-resources.yaml` â€” CPU/memory/storage specs for all services
5. `opentofu/<provider>/<env>/global-values.yaml` â€” environment config (edited by operator)
6. `opentofu/<provider>/<env>/global-cloud-values.yaml` â€” generated by OpenTofu post-provisioning

### Subchart Conditions (install_service pattern)

Subcharts in `Chart.yaml` have `condition: <chart>.enabled` flags. The `install_service` function exploits this to deploy a single subchart by setting `--set <target>.enabled=true` and all others to `false`. This is the correct way to redeploy a single service without running the full bundle.

### Infrastructure (OpenTofu)

- Azure: `opentofu/azure/modules/` â€” AKS, VNet, Storage, Key Vault, Managed Identity, Workload Identity
- GCP: `opentofu/gcp/modules/` â€” equivalent GKE modules
- Terragrunt orchestrates module dependency order
- State backend initialized separately via `create_tf_backend.sh` before first `terragrunt run --all apply`

### Key Databases

- **YugabyteDB**: Primary DB for all building blocks â€” CQL (9042) and YSQL/PostgreSQL (5433)
- **Elasticsearch**: Full-text search for content and learning data
- **JanusGraph**: Graph DB backed by YugabyteDB CQL, used by KnowledgeBB
- **Kafka**: KRaft mode (no ZooKeeper), event streaming across all services

### Scripts (Dockerized)

`scripts/` contains Dockerized utilities that run as Kubernetes Jobs:
- `kong-api-scripts/` â€” Kong route/plugin configuration (`kong_apis.py`, `common.py`)
- `sunbird-yugabyte-migrations/` â€” DB schema migrations
- `keycloak-21.1.2/` â€” Keycloak realm/client setup
- `janusgraph/` â€” JanusGraph index initialization

These are built and pushed via `.github/workflows/build-push-images.yml` and referenced in `helmcharts/images.yaml`.

## Configuration

`global-values.yaml` (the main operator-edited file) contains:
- Cloud provider credentials and region
- Domain name and SSL certificates
- Google OAuth, ReCAPTCHA keys
- Email SMTP settings
- Feature flags: `deployed_dial_addon`, `enable_asset_enrichment`

Never commit actual credentials. The template contains placeholder values to replace.

## Required CLI Tools

`jq`, `yq`, `rclone`, `openssl`, `kubectl`, `helm`, `postman` (CLI), `opentofu` v1.11.4, `terragrunt` v0.77.5, cloud CLI (`az` or `gcloud`).

## Addons

Optional extensions under `addons/`: `dial` (QR code content), `discussion-forum` (NodeBB), `video-stream-generator` (HLS transcoding). Each has its own Helm chart and `addon.sh` install/uninstall script. Addons share `addons/global-values.yaml` and `addons/images.yaml`.
