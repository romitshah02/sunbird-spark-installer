# Sunbird Addons

## Overview

Addons are optional services that can be deployed to an existing Sunbird cluster to extend its functionality. Each addon is independently deployable and manageable.

## Available Addons

### DIAL Service

QR code generation and management service for learning content. Provides DIAL code creation, linking, and management capabilities.

**Documentation**: [addons/dial/README.md](dial/README.md)

**Quick Install**:
```bash
cd addons/dial
./script/manage.sh install
```

### Video Stream Generator

Apache Flink job for video streaming and media processing. Handles video transcoding and HLS stream generation using Azure Media Services or AWS Elemental MediaConvert.

**Documentation**: [addons/video-stream-generator/README.md](video-stream-generator/README.md)

**Quick Install**:
```bash
cd addons/video-stream-generator
./script/manage.sh install
```

## Architecture

Each addon follows a consistent structure:

```
addons/
├── global-values.yaml           # Shared global values for all addons
└── <addon-name>/
    ├── README.md                # Addon-specific documentation
    ├── helmchart/               # Parent directory for helm chart
    │   └── <addon-name>/        # Actual Helm chart directory
    │       ├── Chart.yaml
    │       ├── values.yaml     # Chart-specific values
    │       ├── templates/      # Kubernetes manifests
    │       └── configs/        # Application configuration files (if needed)
    ├── opentofu/                # OpenTofu/Terraform configurations (optional)
    └── script/
        └── manage.sh            # Installation/uninstallation script
```

**Note**: The `opentofu/` directory is optional and used when an addon requires additional cloud resources (e.g., storage accounts, databases, networking) to be provisioned. If present, run the OpenTofu configurations before installing the addon's Helm chart.

## Configuration System

Addons use a layered configuration approach:

1. **Base Global Values** (`opentofu/azure/template/global-values.yaml` or `opentofu/gcp/template/global-values.yaml`)
   - Contains cloud provider configurations, storage endpoints, etc.

2. **Cloud-Generated Values** (`opentofu/azure/template/global-cloud-values.yaml` or `opentofu/gcp/template/global-cloud-values.yaml`)
   - Generated by OpenTofu after infrastructure provisioning
   - Contains dynamic values like storage keys, endpoints, containers etc.

3. **Addon Global Values** (`addons/global-values.yaml`)
   - Shared configuration for all addons (service endpoints, shared settings)
   - Contains connections to Elasticsearch, Redis, Kafka, Yugabyte, etc.
   - Values here are merged with cloud-generated values and passed to Helm.

## Prerequisites

Before installing any addon:

- ✅ Kubernetes cluster is running
- ✅ `kubectl` is configured and connected to the cluster
- ✅ `helm` 3.x is installed
- ✅ **OpenTofu has been run** to generate cloud configuration files
- ✅ Main Sunbird services are deployed (Elasticsearch, Redis, Kafka, Yugabyte, etc.)

## General Usage

All addons follow a similar pattern:

### Installation

```bash
# Navigate to the addon directory
cd addons/<addon-name>

# Install with default settings (defaults to azure)
./script/manage.sh install

# Install for a specific cloud provider
./script/manage.sh install azure
./script/manage.sh install gcp
```

### Verification

```bash
# Check pod status
kubectl get pods -n sunbird -l app.kubernetes.io/name=<addon-name>

# Check logs
kubectl logs -n sunbird -l app.kubernetes.io/name=<addon-name> -f

# Describe the deployment
kubectl describe deployment -n sunbird <addon-name>
```

### Uninstallation

```bash
# Navigate to the addon directory
cd addons/<addon-name>

# Uninstall
./script/manage.sh uninstall

```

For detailed installation instructions, troubleshooting, and configuration options, refer to each addon's README file.
